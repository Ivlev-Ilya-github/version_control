### **Разделы команд Git!**
_Эта структурированная организация помогает лучше понять, какие команды использовать для различных задач в Git._
#### Настройка и создание репозиториев:
- `git init` - **Инициализация нового репозитория**
- `git clone [url]` - **Клонирование репозитория**
- `git config --global user.name "[name]"` - **Настройка имени пользователя**
- `git config --global user.email "[email address]"` - **Настройка email пользователя**
#### Основные операции:
- `git add [file]` - **Добавление файла(ов) к коммиту**
- `git commit -m "[message]"` - **Создание коммита с сообщением**
- `git status` - **Проверка статуса изменений**
- `git push [remote] [branch]` - **Отправка изменений в удалённый репозиторий**
- `git pull [remote] [branch]` - **Получение изменений из удалённого репозитория**
- `git diff` - **Просмотр изменений**
#### Ветвление и слияние:
- `git branch` - **Список веток**
- `git checkout [branch-name]` - **Переключение на другую ветку**
- `git merge [branch]` - **Слияние веток**
- `git rebase [branch]` - **Перебазирование ветки**
#### Исправление ошибок:
- `git reset [file]` - **Отмена изменений в файле**
- `git rm [file]` - **Удаление файла из репозитория**
- `git stash` - **Сохранение текущих изменений в стек**
- `git stash pop` - **Применение изменений из стека**
- `git cherry-pick [commit]` - **Применение изменений из другого коммита**
- `git revert [commit]` - **Отмена изменений коммита**
#### Инспекция и сравнение:
- `git log` - **Просмотр истории коммитов**
- `git show [commit]` - **Показать информацию о коммите**
- `git tag [tag-name]` - **Создание тега для указания важных моментов**
#### Дополнительные команды:
- `git fetch [remote]` - **Загрузка информации из удалённого репозитория**
- `git remote add [shortname] [url]` - **Добавление нового удалённого репозитория**
- `git remote remove [shortname]` - **Удаление удалённого репозитория**
- `git remote rename [old-shortname] [new-shortname]` - **Переименование удалённого репозитория**
- `git rebase --interactive [base]` - **Интерактивное перебазирование**
- `git gc` - **Очистка ненужных файлов и оптимизация локального репозитория**
- `git fsck` - **Проверка целостности файловой системы Git**
- `git archive [branch]` - **Архивация репозитория**
- `git bisect start` - **Использование двоичного поиска для обнаружения багов**
